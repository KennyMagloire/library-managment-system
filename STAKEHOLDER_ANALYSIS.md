# Stakeholder Analysis

---

## Librarian

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Manages books, members, and borrowing/return transactions |
| **Key Concerns**| Fast and accurate operations, ease of use, minimal errors |
| **Pain Points** | Manual record-keeping, repetitive tasks, tracking overdue books |
| **Success Metrics** | Faster transaction processing, reduced errors, improved efficiency |

---

## Library Member (Student/User)

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Borrows and returns books through the librarian |
| **Key Concerns**| Book availability, due dates, fines, accurate records |
| **Pain Points** | Unclear book availability, forgetting due dates, accumulating fines |
| **Success Metrics** | Easy access to information, fewer overdue fines, accurate borrowing history |

---

## System Administrator (IT Support)

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Maintains the system, manages backups, ensures security and availability |
| **Key Concerns**| System stability, data integrity, security, backup and recovery |
| **Pain Points** | Difficulty restoring data, lack of monitoring tools, troubleshooting issues |
| **Success Metrics** | Successful backups, minimal downtime, efficient issue resolution |

---

## Head Librarian

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Oversees library staff and ensures proper system usage |
| **Key Concerns**| Monitoring staff activities, accountability, access to reports |
| **Pain Points** | Lack of visibility into staff actions, difficulty detecting misuse |
| **Success Metrics** | Access to system logs, improved control, better staff supervision |

---

## University Administration

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Evaluates library performance and supports decision-making |
| **Key Concerns**| System efficiency, usage statistics, resource management |
| **Pain Points** | Lack of reliable data, limited insight into system usage |
| **Success Metrics** | Accurate reports, improved decision-making, better resource allocation |

---

## External Auditor / Compliance Officer

| Attribute        | Description |
|-----------------|------------|
| **Role**        | Ensures compliance, audits system records and operations |
| **Key Concerns**| Data integrity, transparency, traceability of actions |
| **Pain Points** | Missing logs, incomplete records, difficulty verifying transactions |
| **Success Metrics** | Complete audit trails, easy access to logs, verified compliance |
