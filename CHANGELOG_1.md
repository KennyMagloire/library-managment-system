## [0.5.0] - 2026-05-03

### Added

#### Task 1 — Class Implementation (`/src/main/java/com/library`)

- `Book.java` — Book entity with ISBN, title, author, genre, publicationYear, totalCopies, availableCopies. Implements `Cloneable` to support the Prototype pattern. Stock management handled via `decreaseCopies()` and `increaseCopies()` methods.
- `Patron.java` — Patron entity with patronId, institutionId, name, email, phone, affiliationType, registrationDate, activeLoans, and a list of loans. `hasOutstandingFines()` loops through loans to check for unpaid fines.
- `Loan.java` — Transaction record linking a Book and a Patron. Checkout date and due date (14 days) set automatically via `LocalDate`. Fine calculation uses `ChronoUnit.DAYS.between()`.
- `Fine.java` — Fine entity with composition relationship to Loan. Fine cannot exist without its parent Loan.
- `Report.java` — Report entity generated by authorised SystemUsers only (HeadLibrarian, UniversityAdministrator).
- `AuditLog.java` — Records every state-changing action with timestamp and userId. Uses static list to maintain a single shared log across all instances.
- `SystemUser.java` — Core actor class with authentication, role-based permission checking, and all operational methods. Every method logs to AuditLog.
- `Main.java` — Entry point demonstrating all six creational patterns running together.

#### Enums
- `BookStatus` — AVAILABLE, BORROWED, LOST, DAMAGED
- `AffiliationType` — STUDENT, LECTURER, STAFF
- `LoanStatus` — ACTIVE, RETURNED, OVERDUE
- `FineStatus` — UNPAID, PAID, WAIVED
- `UserRole` — LIBRARIAN, HEAD_LIBRARIAN, SYSTEM_ADMIN, UNIVERSITY_ADMINISTRATOR, EXTERNAL_AUDITOR
- `ReportType` — CIRCULATION, FINES, MEMBER_ACTIVITY, AUDIT
- `Permission` — GENERATE_REPORT, MANAGE_USERS, WAIVE_FINE, GENERAL
- `AuditAction` — LOGIN, LOGOUT, REGISTER_PATRON, CHECKOUT, RETURN, ADD_BOOK, REMOVE_BOOK, GENERATE_REPORT, CHANGE_PASSWORD, WAIVE_FINE

#### Task 2 — Creational Patterns (`/src/main/java/com/library/patterns`)

- `singleton/DatabaseConnection.java` — Thread-safe Singleton using double-checked locking with `volatile`. Manages single MySQL connection via JDBC.
- `simplefactory/SystemUserFactory.java` — Creates SystemUser instances based on UserRole enum. Centralises all user creation.
- `factorymethod/ReportFactory.java` — Interface defining the factory method contract.
- `factorymethod/CirculationReportFactory.java` — Creates CIRCULATION reports.
- `factorymethod/FineReportFactory.java` — Creates FINES reports.
- `factorymethod/AuditReportFactory.java` — Creates AUDIT reports.
- `factorymethod/MemberActivityReportFactory.java` — Creates MEMBER_ACTIVITY reports.
- `abstractfactory/NotificationFactory.java` — Abstract factory interface.
- `abstractfactory/ConsoleNotificationFactory.java` — Console notification family for development/testing.
- `abstractfactory/EmailNotificationFactory.java` — Email notification family for production.
- `abstractfactory/ConsoleBookNotification.java` — Console book notifications.
- `abstractfactory/ConsolePatronNotification.java` — Console patron notifications.
- `abstractfactory/EmailBookNotification.java` — Email book notifications.
- `abstractfactory/EmailPatronNotification.java` — Email patron notifications.
- `builder/PatronBuilder.java` — Builds Patron objects step by step. Required fields in constructor, optional fields via fluent setters.
- `prototype/BookPrototypeCache.java` — Stores and clones pre-configured Book templates (TEXTBOOK, FICTION, REFERENCE).

#### Task 3 — Unit Tests (`/src/test/java/com/library`)

- `SingletonTest.java` — Tests single instance, non-null instance, and thread safety.
- `SimpleFactoryTest.java` — Tests correct user type creation and null input handling.
- `BuilderTest.java` — Tests required fields, optional fields, and invalid input edge cases.
- `PrototypeTest.java` — Tests clone independence, category matching, and invalid key handling.
- `FactoryMethodTest.java` — Tests each factory produces the correct report type.
- `AbstractFactoryTest.java` — Tests each factory produces the correct notification family.

### Changed
- `Book.java` — Added `totalCopies` and `availableCopies` fields. Status now controlled internally by `decreaseCopies()` and `increaseCopies()` instead of a public setter.
- `Patron.java` — Renamed `studentNumber` to `institutionId` to support all affiliation types. Added `List<Loan> loans` field. `registrationDate` now set automatically via `LocalDate.now()`.
- `ReportGenerator.java` — Refactored into abstract `Report` class to support Factory Method pattern.

### Design Decisions
- Used enums throughout instead of plain Strings for all status and type fields to prevent typos and enforce compile-time safety.
- `institutionId` replaces `studentNumber` because Patrons can be students, lecturers, or staff — asking a lecturer for a student number makes no sense.
- `LocalDate` and `LocalDateTime` used for all date fields to enable real date arithmetic.
- Composition between `Loan` and `Fine` — Fine holds a direct reference to its parent Loan and cannot exist independently.
